### Formulários e Entrada de Dados

- [ ] Campos de formulário devem possuir rótulos visíveis e compreensíveis, indicando claramente a informação esperada. [NBR 5.9.1] [WCAG 2.2 - 3.3.2 Rótulos ou Instruções]

- [ ] Os rótulos devem estar programaticamente associados aos seus respectivos campos, permitindo identificação por leitores de tela. [NBR 5.9.3] [WCAG 2.2 - 1.3.1 Informação e Relações, 4.1.2 Nome, Função, Valor]

- [ ] Campos obrigatórios devem ser claramente identificados, sem depender somente de cor ou símbolo sem explicação. [NBR 5.9.7] [WCAG 2.2 - 3.3.2 Rótulos ou Instruções]

- [ ] O tipo de dado esperado deve ser determinado quando aplicável, como e-mail, telefone, data ou número. [NBR 5.9.8] [WCAG 2.2 - 1.3.5 Identificar o Propósito da Entrada de Dados]

- [ ] Mensagens de erro devem ser descritivas, indicando o problema ocorrido de forma objetiva. [NBR 5.9.9] [WCAG 2.2 - 3.3.1 Identificação de Erro]

- [ ] Quando possível, a interface deve apresentar sugestão de correção para o erro identificado, como “Digite um e-mail válido”. [NBR 5.9.10] [WCAG 2.2 - 3.3.3 Sugestão de Erro]

- [ ] Em formulários críticos, devem existir mecanismos de prevenção de erro, como confirmação, revisão ou possibilidade de correção antes do envio final. [NBR 5.9.12] [WCAG 2.2 - 3.3.4 Prevenção de Erros (Jurídicos, Financeiros, de Dados)]

- [ ] Se o usuário já tiver inserido uma informação em um passo anterior do processo (como endereço de entrega), o sistema deve preencher os próximos campos redundantes automaticamente ou oferecer uma forma fácil de resgatar esse dado, reduzindo a fadiga cognitiva e o esforço de digitação. [WCAG 2.2 - 3.3.7 Entrada de Dados Redundante]

### Feedback, Previsibilidade e Recuperação de Erros

- [ ] A interface deve fornecer feedback claro sobre ações realizadas pelo usuário, como envio, carregamento, sucesso ou erro. [NBR 5.13.8] [WCAG 2.2 - 4.1.3 Mensagens de Status]

- [ ] Mudanças de contexto ao receber foco devem ser previsíveis, evitando que a interface mude de tela ou execute ações inesperadas apenas porque um elemento recebeu foco. [NBR 5.8.9] [WCAG 2.2 - 3.2.1 Ao Receber Foco]

- [ ] Mudanças de contexto após entrada de dados devem ser previsíveis e compreensíveis para o usuário. [NBR 5.8.10] [WCAG 2.2 - 3.2.2 Na Entrada de Dados]

- [ ] Componentes customizados devem possuir semântica, estados, propriedades e valores acessíveis, garantindo que tecnologias assistivas compreendam seu funcionamento. [NBR 5.13.12] [NBR 5.13.13] [WCAG 2.2 - 4.1.2 Nome, Função, Valor]
