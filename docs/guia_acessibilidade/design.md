### Usabilidade, IHC e Acessibilidade

A usabilidade deve permitir que o usuário atinja seus objetivos de forma eficaz, eficiente e satisfatória; quando a interface cria barreiras para pessoas com deficiência, ela falha também como experiência de uso. [[NBR 5.1.13]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

A interface deve ser operável, compreensível e previsível, pois a IHC busca reduzir erros, aumentar eficiência e melhorar a satisfação do usuário. [[NBR 5.7.15]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

A acessibilidade digital deve ser tratada como parte central da qualidade da interface, não como funcionalidade opcional. [[NBR 4]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

---

### Design Visual, Contraste e Cores

A interface não deve depender apenas da cor para transmitir informações, como erro, sucesso, alerta ou estado selecionado. [[NBR 5.11.1]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

Textos devem possuir contraste mínimo suficiente em relação ao fundo, garantindo legibilidade para usuários com baixa visão. [[NBR 5.11.3]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

Componentes de interface, como botões, campos e bordas, devem ter contraste adequado para serem percebidos visualmente. [[NBR 5.11.4]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

Objetos gráficos importantes, como ícones, indicadores e elementos informativos, devem possuir contraste suficiente. [[NBR 5.11.5]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

O indicador de foco também deve possuir contraste perceptível, para que o usuário consiga acompanhar a navegação por teclado. [[NBR 5.11.6]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

---

### Apresentação, Layout e Responsividade

A ordem de apresentação dos elementos deve preservar o significado do conteúdo, mantendo uma sequência lógica para leitura e interação. [[NBR 5.10.2]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

O design deve ser responsivo, funcionando adequadamente em diferentes tamanhos de tela e dispositivos. [[NBR 5.10.4]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

A interface não deve depender apenas de características sensoriais, como posição, formato, tamanho ou cor, para explicar uma ação. [[NBR 5.10.1]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

O zoom não deve ser bloqueado, permitindo que o usuário amplie o conteúdo conforme sua necessidade. [[NBR 5.13.5]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

O texto deve poder ser redimensionado sem perda de conteúdo ou funcionalidade. [[NBR 5.12.7]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

---

### Relação com “Bad vs Good UI Design”

Uma interface “good UI” deve tornar as ações visíveis, compreensíveis e previsíveis, reduzindo a necessidade de instruções excessivas para tarefas simples. [[NBR 5.8.3]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

A consistência visual e funcional contribui para que o usuário reconheça padrões de navegação e interação ao longo da interface. [[NBR 5.7.15]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

A visibilidade do estado do sistema deve ser garantida por mensagens, foco visível, feedback de ações e identificação clara dos componentes. [[NBR 5.1.1]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR) [[NBR 5.13.8]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)

O bom design não deve apenas “parecer bonito”; ele deve permitir uso eficiente, acessível e seguro para diferentes perfis de usuários. [[NBR 5.10.4]](https://unb-ihc.github.io/IHC-2025.2-Grupo6/manual/devWeb/#NBR)
