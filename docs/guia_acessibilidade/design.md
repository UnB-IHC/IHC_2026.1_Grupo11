### Usabilidade, IHC e Acessibilidade

- [ ] A usabilidade deve permitir que o usuário atinja seus objetivos de forma eficaz, eficiente e satisfatória; quando a interface cria barreiras para pessoas com deficiência, ela falha também como experiência de uso. [NBR 5.1.13](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - Princípios Básicos](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#consistent-navigation)

- [ ] A interface deve ser operável, compreensível e previsível, pois a IHC busca reduzir erros, aumentar eficiência e melhorar a satisfação do usuário. [NBR 5.7.15](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - 3.2 Previsível](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#consistent-navigation:~:text=Diretriz%203.2%20Previs%C3%ADvel)

- [ ] A acessibilidade digital deve ser tratada como parte central da qualidade da interface, não como funcionalidade opcional. [NBR 4](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - Introdução](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#consistent-navigation)

- [ ] Processos de login e autenticação não devem forçar o usuário a resolver testes cognitivos complexos (como transcrever textos distorcidos de imagens ou memorizar dados), permitindo métodos alternativos seguros, como preenchimento automático por gerenciadores de senha ou biometria. [WCAG 2.2 - 3.3.8 Autenticação Acessível (Mínimo)](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#consistent-navigation:~:text=Crit%C3%A9rio%20de%20Sucesso%203.3.8%20Autentica%C3%A7%C3%A3o%20Acess%C3%ADvel%20(M%C3%ADnimo))

### Design Visual, Contraste e Cores

- [ ] A interface não deve depender apenas da cor para transmitir informações, como erro, sucesso, alerta ou estado selecionado. [NBR 5.11.1](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - 1.4.1 Uso de Cor](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#use-of-color:~:text=Crit%C3%A9rio%20de%20Sucesso%201.4.1%20Uso%20de%20Cores)

- [ ] Textos devem possuir contraste mínimo suficiente em relação ao fundo, garantindo legibilidade para usuários com baixa visão. [NBR 5.11.3](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - 1.4.3 Contraste (Mínimo)](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#use-of-color:~:text=Crit%C3%A9rio%20de%20Sucesso%201.4.3%20Contraste%20(M%C3%ADnimo))

- [ ] Componentes de interface, como botões, campos e bordas, devem ter contraste adequado para serem percebidos visualmente. [NBR 5.11.4](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - 1.4.11 Contraste Não Textual](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#use-of-color:~:text=Crit%C3%A9rio%20de%20Sucesso%201.4.11%20Contraste%20N%C3%A3o%20textual)

- [ ] Objetos gráficos importantes, como ícones, indicadores e elementos informativos, devem possuir contraste suficiente. [NBR 5.11.5](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - 1.4.11 Contraste Não Textual](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#use-of-color:~:text=Crit%C3%A9rio%20de%20Sucesso%201.4.11%20Contraste%20N%C3%A3o%20textual)

- [ ] O indicador de foco também deve possuir contraste perceptível, para que o usuário consiga acompanhar a navegação por teclado. [NBR 5.11.6](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - 1.4.11 Contraste Não Textual, 2.4.13 Aparência do Foco](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#use-of-color:~:text=Crit%C3%A9rio%20de%20Sucesso%201.4.11%20Contraste%20N%C3%A3o%20textual)

### Apresentação, Layout e Responsividade

- [ ] A ordem de apresentação dos elementos deve preservar o significado do conteúdo, mantendo uma sequência lógica para leitura e interação. [NBR 5.10.2](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - 1.3.2 Sequência com Significado](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#use-of-color:~:text=Crit%C3%A9rio%20de%20Sucesso%201.3.2%20Sequ%C3%AAncia%20com%20Significado)

- [ ] O design deve ser responsivo, funcionando adequadamente em diferentes tamanhos de tela e dispositivos. [NBR 5.10.4](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - 1.4.10 Refluxo](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#use-of-color:~:text=Crit%C3%A9rio%20de%20Sucesso%201.4.10%20Realinhar)

- [ ] A interface não deve depender apenas de características sensoriais, como posição, formato, tamanho ou cor, para explicar uma ação. [NBR 5.10.1](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - 1.3.3 Características Sensoriais](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#use-of-color:~:text=Crit%C3%A9rio%20de%20Sucesso%201.3.3%20Caracter%C3%ADsticas%20Sensoriais)

- [ ] O zoom não deve ser bloqueado, permitindo que o usuário amplie o conteúdo conforme sua necessidade. [NBR 5.13.5](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - 1.4.4 Redimensionamento de Texto](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#use-of-color:~:text=Crit%C3%A9rio%20de%20Sucesso%201.4.4%20Redimensionar%20Texto)

- [ ] O texto deve poder ser redimensionado sem perda de conteúdo ou funcionalidade. [NBR 5.12.7](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - 1.4.4 Redimensionamento de Texto](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#use-of-color:~:text=Crit%C3%A9rio%20de%20Sucesso%201.4.4%20Redimensionar%20Texto)

### Relação com “Bad vs Good UI Design”

- [ ] Uma interface “good UI” deve tornar as ações visíveis, compreensíveis e previsíveis, reduzindo a necessidade de instruções excessivas para tarefas simples. [NBR 5.8.3](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - 3.3.2 Rótulos ou Instruções](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#use-of-color:~:text=Crit%C3%A9rio%20de%20Sucesso%203.3.2%20R%C3%B3tulos%20ou%20Instru%C3%A7%C3%B5es)

- [ ] A consistência visual e funcional contribui para que o usuário reconheça padrões de navegação e interação ao longo da interface. [NBR 5.7.15](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - 3.2.3 Navegação Consistente, 3.2.4 Identificação Consistente](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#use-of-color:~:text=Crit%C3%A9rio%20de%20Sucesso%203.3.2%20R%C3%B3tulos%20ou%20Instru%C3%A7%C3%B5es)

- [ ] A visibilidade do estado do sistema deve ser garantida por mensagens, foco visível, feedback de ações e identificação clara dos componentes. [NBR 5.1.1](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [NBR 5.13.8](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - 4.1.3 Mensagens de Status](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#use-of-color:~:text=Crit%C3%A9rio%20de%20Sucesso%204.1.3%20Mensagens%20de%20Status)

- [ ] O bom design não deve apenas “parecer bonito”; ele deve permitir uso eficiente, acessível e seguro para diferentes perfis de usuários. [NBR 5.10.4](https://mwpt.com.br/wp-content/uploads/2025/04/ABNT-NBR-17225-Acessibilidade-Digital.pdf) [WCAG 2.2 - Princípio 4 Robusto](https://www.w3c.br/traducoes/wcag/wcag22-pt-BR/#intro)
